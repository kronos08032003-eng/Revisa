1. Comandos Linux — Revisão e Guia Prático

Resumo: comandos essenciais para navegação, gestão de arquivos, permissões, processos, rede e scripts.

Navegação e listagem

$ pwd
$ ls -la
$ cd /caminho/para/pasta
$ tree (se instalado)

Dica: combine ls -la com --color=auto para destacar tipos de arquivo.

Gerenciamento de arquivos

$ cp origem destino
$ mv arquivo pasta/
$ rm arquivo.txt
$ rm -rf pasta/  # CUIDADO

Permissões e propriedade

$ ls -l
$ chmod 644 arquivo.txt
$ chmod +x script.sh
$ chown usuario:grupo arquivo

Use notação octal (ex: 755) ou simbólica (u+rwx).

Processos e monitoramento

$ ps aux | grep nome
$ top
$ htop (se instalado)
$ kill PID
$ kill -9 PID

Rede

$ ip addr show
$ ss -tuln
$ ping exemplo.com
$ curl -I https://exemplo.com

Busca e manipulação de texto

$ grep -R "termo" ./
$ find . -name "*.js"
$ awk '{print $1}' arquivo.txt
$ sed -n '1,10p' arquivo.txt

Exemplo poderoso: find . -name '*.log' | xargs grep -i erro

Scripts básicos (bash)

#!/usr/bin/env bash
set -euo pipefail
for f in *.txt; do
  echo "Processando $f"
done

Use set -euo pipefail para scripts mais seguros.

2. Git — Comandos essenciais e fluxos de trabalho

Do controle local ao trabalho colaborativo: commits, branches, merges, rebase e resolução de conflitos.

Configuração inicial

$ git config --global user.name "Seu Nome"
$ git config --global user.email "seu@exemplo.com"
$ git config --global core.editor "code --wait"

Fluxo básico

$ git init
$ git add .
$ git commit -m "Mensagem"
$ git branch -M main
$ git remote add origin git@github.com:usuario/repositorio.git
$ git push -u origin main

Branches, merge e rebase

$ git checkout -b feature/minha-feature
# trabalhar...
$ git add . && git commit -m "feat: adicionar X"
$ git checkout main
$ git pull origin main
$ git merge feature/minha-feature
# ou
$ git rebase main (na branch feature)

Rebase reescreve histórico — use em branches locais antes de compartilhar.

Resolução de conflitos

# após um merge com conflito
$ git status
# editar arquivos conflitantes
$ git add arquivo
$ git commit

Histórico e revisão

$ git log --oneline --graph --decorate
$ git show HEAD~2:README.md
$ git diff origem..destino

Dicas avançadas

  • Use git stash para mudanças temporárias.
  • git cherry-pick para aplicar commits específicos.
  • Configure hooks (pre-commit, pre-push) para qualidade automática.

  • Quer deploy direto do GitHub sem configuração? GitHub Pages.
  • Requisitos empresariais e conformidade? AWS + CI/CD corporativo.

4. Explorando HTTPS e DNS

Como funcionam e como configurar: fluxo da conexão HTTPS, registros DNS e certificados TLS/SSL.

Conceitos básicos de DNS

  • A: aponta para IPv4.
  • AAAA: aponta para IPv6.
  • CNAME: alias para outro nome canônico.
  • TXT: verificações (SPF, DKIM) e validação.
  • NS: delega zona a nameservers.

Como funciona HTTPS (resumo)

  1. Cliente inicia conexão TLS.
  2. Servidor envia certificado X.509.
  3. Cliente valida certificado (CA, nome, validade).
  4. Negociação de parâmetros criptográficos.
  5. Comunicação cifrada com chave de sessão.

Let's Encrypt usa ACME para emitir certificados gratuitos. Plataformas (Netlify, Vercel, Cloudflare) cuidam da emissão e renovação automaticamente.

Configuração típica — passo a passo

# 1. Registrar domínio
# 2. Criar registros DNS (A/AAAA para ips ou CNAME para o provedor)
# 3. Adicionar domínio no painel do provedor de hospedagem
# 4. Verificar domínio (TXT/CNAME) e ativar HTTPS
# 5. Testar: curl -I https://seu-dominio.com

Problemas comuns

  • Certificado inválido: relógio incorreto ou CNAME mal configurado.
  • DNS não propagado: aguardar TTL ou usar dig +trace.
  • Mixed content: recursos HTTP em página HTTPS — atualizar URLs.

5. Exemplo prático: workflow para publicar um site estático

  1. Inicialize repositório Git e crie index.html.
  2. Faça commits e publique no GitHub.
  3. Conecte o repositório ao Vercel/Netlify/Cloudflare Pages.
  4. Configure domínio no painel do provedor.
  5. Ative HTTPS automático e teste o site.
Comandos de exemplo 
$ git init
$ echo "<h1>Olá mundo</h1>" > index.html
$ git add .
$ git commit -m "chore: site inicial"
$ git remote add origin git@github.com:usuario/site.git
$ git push -u origin main

6. Recursos rápidos e boas práticas

  • Use CI para testes e lint antes do deploy.
  • Adote branch protection e revisão por pull request.
  • Automatize renovação de certificados (provedores já fazem).
  • Monitore uptime (UptimeRobot, Pingdom) e performance.
  • Documente o processo no README.md.
